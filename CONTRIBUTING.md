Contributing to ELAN Programming Language

Thank you for your interest in contributing to the ELAN Programming Language project.

ELAN is an open-source programming language designed for modern software development with a powerful standard library and simple syntax.

How to Contribute

You can contribute in the following ways:

1. Bug Reports

If you find a bug:

1. Open an issue in the GitHub repository
2. Clearly describe the problem
3. Include example ELAN code if possible

---

2. Feature Requests

You may suggest new features such as:

- New standard libraries
- Compiler improvements
- Runtime performance improvements
- Documentation improvements

---

3. Pull Requests

Steps to contribute code:

1. Fork the repository
2. Create a new branch

Example:

git checkout -b feature-new-library

3. Make your changes
4. Commit your changes

git commit -m "Add new ELAN library"

5. Push to your fork

git push origin feature-new-library

6. Create a Pull Request on GitHub

---

Code Guidelines

Please follow these rules:

• Keep code simple and readable
• Follow ELAN coding style
• Add comments for complex logic
• Do not break existing features

---

Standard Library Contributions

When adding a new library:

1. Place it inside:

stdlib/

2. Example file name:

elanmath.el

3. Include documentation comments at the top of the file.

---

Issue Labels

Common issue labels:

- "bug"
- "enhancement"
- "documentation"
- "stdlib"
- "compiler"

---

Code of Respect

All contributors should:

• Be respectful
• Encourage open discussion
• Help improve the ELAN ecosystem
